# AWS Cloud Computing: The 20% You Need to Know

## 1. What is AWS?
- Cloud computing platform provided by Amazon
- Offers 200+ fully featured services from data centers globally
- Market leader in cloud infrastructure services
- Supports businesses of all sizes, from startups to enterprises

## 2. Core Cloud Computing Concepts

### 2.1 Cloud Service Models
- Infrastructure as a Service (IaaS)
- Platform as a Service (PaaS)
- Software as a Service (SaaS)

### 2.2 Deployment Models
- Public Cloud
- Private Cloud
- Hybrid Cloud
- Multi-Cloud

## 3. Fundamental AWS Services

### 3.1 Compute Services
#### Amazon EC2 (Elastic Compute Cloud)
- Virtual servers in the cloud
- Scalable computing capacity
- Key features:
  - Multiple instance types
  - Easy scaling
  - Pay only for used resources

#### AWS Lambda
- Serverless compute service
- Run code without provisioning servers
- Event-driven execution
- Supports multiple programming languages

### 3.2 Storage Services
#### Amazon S3 (Simple Storage Service)
- Object storage service
- Highly scalable and durable
- Use cases:
  - Website hosting
  - Backup and archiving
  - Data lakes

#### Amazon EBS (Elastic Block Store)
- Persistent block storage volumes
- Designed for EC2 instances
- Different volume types for various performance needs

### 3.3 Networking Services
#### Amazon VPC (Virtual Private Cloud)
- Isolated cloud resources
- Custom network configuration
- Security and connectivity controls

#### Amazon Route 53
- Scalable Domain Name System (DNS)
- Route end-user requests
- Health checking of resources

## 4. Database Services

### 4.1 Amazon RDS (Relational Database Service)
- Managed relational databases
- Supports multiple engines:
  - MySQL
  - PostgreSQL
  - Oracle
  - SQL Server
  - MariaDB

### 4.2 Amazon DynamoDB
- Fully managed NoSQL database
- Seamless scalability
- Low-latency performance
- Ideal for mobile, web, gaming, ad-tech applications

## 5. Security and Identity Services

### 5.1 AWS IAM (Identity and Access Management)
- Manage access to AWS services
- Create users, groups, roles
- Implement fine-grained permissions
- Multi-factor authentication

### 5.2 AWS Security Best Practices
- Use least privilege principle
- Enable encryption
- Regular security audits
- Implement network isolation

## 6. Monitoring and Management

### 6.1 Amazon CloudWatch
- Monitoring and observability service
- Collect metrics and logs
- Set alarms
- Automatic actions

### 6.2 AWS CloudTrail
- Governance, compliance, and audit service
- Track user activity and API usage
- Log file management

## 7. Containerization and Orchestration

### 7.1 Amazon ECS (Elastic Container Service)
- Container orchestration service
- Run and scale Docker containers

### 7.2 Amazon EKS (Elastic Kubernetes Service)
- Managed Kubernetes service
- Run Kubernetes on AWS
- Simplified cluster management

## 8. Serverless Computing

### 8.1 AWS Serverless Application Model (SAM)
- Framework for building serverless applications
- Simplified deployment
- Local testing and debugging
